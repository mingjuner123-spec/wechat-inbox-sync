# Obsidian 内容同步助手运营后台

这是独立的用户与 Pro 管理后台，不再依赖小程序里的 `pages/admin/index`。

## 功能

- 总览：访问、到达绑定页、绑定成功、首次同步、二次同步、Pro 用户漏斗
- 运营漏斗：查看转化率、掉点诊断、新老用户分层、最近 14 天趋势
- 用户管理：查看每个用户的绑定、同步、Pro 状态
- Pro 管理：查看 Pro 用户、到期时间、续期、启用、禁用
- 兑换码：生成兑换码、标记发放、撤回发放、启用、禁用
- 绑定设备：查看绑定码和已绑定电脑数量

## 运营漏斗口径

后台核心看这条线：

```text
访问数 → 到达绑定页面数 → 绑定成功数 → 首次同步成功数 → 二次以上同步用户数 → Pro 用户数
```

- `访问数`：打开过小程序的去重用户。
- `到达绑定页面数`：进入过绑定 Obsidian 页面或触发绑定页曝光的去重用户。
- `绑定成功数`：插件端绑定成功的去重用户；同时会合并已经绑定设备的绑定码数据。
- `首次同步成功数`：至少有 1 条素材被插件同步到 Obsidian 的用户。
- `二次以上同步用户数`：至少同步过 2 条素材的用户，用来看复用。
- `Pro 用户数`：当前仍在有效期内的 Pro 用户。

建议日常这样看：

- `总览`：看今天是否有同步异常、云端暂存积压、Pro 到期提醒。
- `运营漏斗`：看哪个环节掉人最多，优先优化那个环节。
- `用户管理`：查单个用户卡在哪一步。
- `Pro 管理`：处理续期、禁用、恢复。
- `兑换码`：标记哪些码已发、未发、已激活。
- `绑定设备`：排查绑定码和多电脑绑定问题。

## 部署步骤

1. 在微信开发者工具里上传并部署云函数：

   ```text
   cloudfunctions/adminApi
   cloudfunctions/syncApi
   ```

   部署方式选择“上传并部署：云端安装依赖”。

2. 确认云函数环境变量里有：

   ```text
   REDEEM_ADMIN_SECRET=你的管理密钥
   ```

3. 后台复用现有的 `syncApi` HTTP 访问路径，不需要额外新增 `/admin-api` 路由。

   当前环境已经有同步接口：

   ```text
   https://he02-d8gebzv050ed6c4ef-d350b93bf-1357443479.ap-shanghai.app.tcloudbase.com/sync
   ```

   后台管理接口会走：

   ```text
   https://he02-d8gebzv050ed6c4ef-d350b93bf-1357443479.ap-shanghai.app.tcloudbase.com/sync/admin/summary
   ```

4. 打开 `admin-console/index.html`。

5. 在顶部输入：

   ```text
   syncApi HTTP 地址
   管理密钥
   ```

6. 点击保存，然后刷新总览。

## 注意

- `访问数` 和 `到达绑定页面数` 从埋点上线后开始累计，历史访问无法自动补回。
- 当前后台每类数据最多读取 5000 条；超过后需要做独立统计汇总表。
- 这个后台只给管理员使用，不要把地址发给用户。
