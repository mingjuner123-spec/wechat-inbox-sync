# 插件绑定误报与小红书图片修复设计

## 目标

基于正式插件 1.3.48 发布 1.3.49：避免把普通 HTTP 403 一律解释为绑定码失效；成功绑定新码后立即使用新绑定；飞书 OAuth 可跳过明确失效的旧绑定；小红书远程图片本地化下载复用平台请求头。

## 范围

只修改 `obsidian-plugin/wechat-inbox-sync/`、插件回归测试、版本元数据和发布日志。不修改小程序、云函数、业务数据、用户绑定记录、保存目录逻辑和本地 OCR 安装器。

## 设计

绑定错误分类只接受服务端明确的失效信号，例如 `Invalid bind code`、`Invalid or expired token`、`绑定码未绑定或已失效`；裸 `403` 不再属于失效信号。请求层仍可对迁移入口做一次兼容重试，但最终错误保留服务端业务信息。成功绑定的新 token 写入主 token 并排到绑定列表首位。飞书 OAuth 在候选绑定明确失效时继续尝试下一项，其他错误立即返回。

远程 Markdown 图片本地化按页面 URL 判断小红书来源。小红书图片下载调用已有 `getXiaohongshuRequestHeaders`，其他网页保持现有行为。下载成功后继续保存为本地附件并替换 Markdown；下载失败仍保留原 URL，同时通过既有 `onError` 回调留下诊断。

## 验证

测试覆盖普通 403 不属于绑定失效、明确失效仍属于绑定失效、新绑定成为主绑定、飞书跳过旧绑定，以及小红书图片本地化向下载层传入 Referer/Cookie/User-Agent。发布门禁覆盖插件主回归、包结构、语法、版本 JSON、ZIP 与 GitHub Release/默认分支一致性。
