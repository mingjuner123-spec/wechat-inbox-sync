# 无效绑定码允许重新输入

## 目标

用户点击“立即绑定”后，只有服务端验证成功的绑定码才能进入已绑定状态。服务端返回无效绑定码时，插件提示“绑定码无效，请重新输入”，并保持输入框可编辑。

## 规则

- 输入内容只作为待验证值，不得提前写入正式绑定列表。
- `/bind` 成功后沿用现有有效绑定流程，不改变绑定数量、Pro 权限或同步规则。
- `/bind` 返回 `403` 或 `Invalid bind code` 时不创建绑定、不锁定输入框、不触发 Pro 自动检查。
- 失败后保留用户刚输入的内容，方便直接修改并再次点击绑定。
- 本次不新增换绑、自动清理或服务端解绑规则。

## 验证

- 回归覆盖“输入兑换码并保存设置后仍不是正式绑定”。
- 回归覆盖“无效绑定请求后输入值仍在、绑定列表为空、可再次提交”。
- 保持现有有效绑定成功测试通过。
