# Obsidian 插件 1.3.56 发布设计

## 目标

从公开仓库权威基线 `main@624fa7e` 发布 `1.3.56`：

- 保留 `1.3.55` 已包含的停止当前转写、macOS ASR 固定 Python/wheelhouse、Windows ASR CPU 兼容与部署器可移植性修复；
- 精确加入 `02734f8` 的停止转写入口常驻补丁；
- 不加入体验 Pro 云端逻辑、Stage1A 本地组件调度、Hermes、分销或其他支线。

## 发布身份

- 仓库：`https://github.com/mingjuner123-spec/wechat-inbox-sync.git`
- 默认分支：运行时解析的 `main`
- 基线：`1.3.55@624fa7eebb6e1c6c09e51c72903a871d2228b168`
- 候选分支：`codex/release-plugin-1.3.56`
- 目标版本：`1.3.56`
- 发布源：`obsidian-plugin/wechat-inbox-sync/`

根目录和插件目录的 `manifest.json`、`versions.json` 必须逐字同步。发布身份以受保护合并后的最终 `origin/main` SHA 为准，不假定候选分支提交与合并提交相同。Release 的独立资产和 ZIP 必须由指向该 SHA 的 tag checkout 首次构建，禁止覆盖同版本 Release。

## 验证和回退

公开仓库必须自带独立发布身份门禁，只依赖 Node 内置模块、Git 和固定 GitHub API，不依赖综合项目私有 Harness。发布前验证插件回归、市场包、版本治理、Windows/macOS 安装器语法、组件 manifest/CDN 与固定发布身份。

受保护合并后先同步到最终 `origin/main` SHA，再创建本地 annotated tag；prepublish 只有在远端 tag 精确不存在、Release endpoint 精确返回 HTTP 404、四份版本元数据一致且本地 tag peeled SHA 与最终 main 相同时才通过。随后才能推送 tag，由工作流首次创建 Release。

发布后以三次独立 UTC 新鲜采样核对默认分支、peeled tag、Release 目标提交和五项独立资产；下载正式 ZIP，验证条目及关键文件字节与 tag checkout 一致，再回读 raw manifest 与 Marketplace。网络错误、解析失败、近似 404、版本或 SHA 漂移全部失败关闭。

若默认分支尚未合并或标签尚未推送，直接停止即可；若 Release 已创建但市场闭环失败，不覆盖同版本，修复后使用新的补丁版本。
