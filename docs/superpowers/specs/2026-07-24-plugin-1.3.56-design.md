# Obsidian 插件 1.3.56 发布设计

## 目标

从公开仓库权威基线 `main@624fa7e` 发布 `1.3.56`：

- 保留 `1.3.55` 已包含的停止当前转写、macOS ASR 固定 Python/wheelhouse、Windows ASR CPU 兼容与部署器可移植性修复；
- 精确加入 `02734f8` 的停止转写入口常驻补丁；
- 不加入体验 Pro 云端逻辑、Stage1A 本地组件调度、Hermes、分销或其他支线。

## 发布身份

- 仓库：`https://github.com/mingjuner123-spec/wechat-inbox-sync.git`
- 默认分支：运行时解析的 `main`
- 基线：`1.3.55@624fa7eebb6e1c6c09e51c72903a871d2228b168`
- 候选分支：`codex/release-plugin-1.3.56`
- 目标版本：`1.3.56`
- 发布源：`obsidian-plugin/wechat-inbox-sync/`

根目录和插件目录的 `manifest.json`、`versions.json` 必须逐字同步。Release 的独立资产和 ZIP 必须来自同一候选提交。

## 验证和回退

发布前验证插件回归、市场包、版本治理、Windows/macOS 安装器语法、组件 manifest/CDN 与固定发布身份。合并默认分支后再创建指向同一提交的标签和 Release。发布后回读默认分支、标签、Release 资产、ZIP、raw manifest 与 Marketplace。

若默认分支尚未合并或标签尚未推送，直接停止即可；若 Release 已创建但市场闭环失败，不覆盖同版本，修复后使用新的补丁版本。
