# 体验 Pro 绑定额度与本地转写可控性设计

日期：2026-07-23  
任务：`plugin-bind-transcription-001`

## 已确认的产品规则

1. 正式 Pro 和体验 Pro 在有效期内，均允许同一 Obsidian 插件绑定最多 3 个微信。
2. 体验 Pro 到期后，第一条绑定继续可用，额外第 2、3 条绑定不能再同步；不删除远端绑定记录，后续重新开通体验/正式 Pro 时自动恢复。
3. 无效绑定码仍显示“绑定码无效”且保持输入框可编辑；有效码但不满足绑定额度，必须显示额度/体验已到期原因。
4. “暂停当前转写”是安全停止：终止当前本地任务、该记录保持待同步、下次同步从头重试；不承诺从中断位置继续。

## 绑定规则

### 配额判定

新增“可用于插件绑定额度的权益”判断：正式 Pro 与 `local_transcription_trial` 均视为有效 Pro。原有正式 Pro 识别保留给支付/正式权益语义，不改变其含义。

绑定 `/bind` 时，按同一 `clientId` 下现有绑定和目标绑定的任一有效体验/正式 Pro 计算额度：有效时上限 3，否则上限 1。

### 到期后的访问判定

普通同步 API 认证时，不仅校验绑定码和客户端是否存在，还校验该绑定是否处于当前额度内：

- 有有效体验/正式 Pro：前三条均可访问；
- 无有效体验/正式 Pro：按该客户端的 `boundAt` 升序保留第一条，后续条目返回明确的“体验 Pro 已到期，额外绑定暂不可用”。

这是动态收敛，不修改或删除绑定数据，因此重新获得体验/正式 Pro 后无需重新绑定。

解除绑定必须绕过上述访问额度收敛，只要原始绑定关系存在就允许用户解除，以免到期后无法清理额外绑定。

### 错误传递

插件对 `/bind` 的 4xx 需要用 Node HTTP 回读 JSON 错误体；只有服务端明确的 `Invalid bind code` 才显示“绑定码无效”。配额错误按内容显示，不使用“绑定电脑名额已满”的旧文案。

## 转写进度与暂停

### 根因

现有脚本先写 `preparing / 0%`，随后同步等待 ffmpeg 分段或 Whisper 单段完成；插件只在日志字段变化时刷新。因此任何原生步骤阻塞都会停在 0%，最长由外层两小时超时兜底。

### 修复

Windows 与 macOS 安装器生成的转写脚本均增加：

- 明确阶段：`preparing`、`segmenting`、`transcribing`、`done`；
- 原生子进程 PID、阶段开始时间、心跳时间；
- 轮询式原生进程等待，在子进程执行时每 5 秒写一次心跳；
- 保持原有分段进度，第一段未完成时展示“正在转写第 1/N 段，已运行 …”，不把正常处理中误导成静态 0%。

插件解析心跳和阶段，按阶段显示文案与已运行时间；如果心跳陈旧，显示“任务可能无响应，可暂停后重试”而非继续宣称正常。

### 可见停止入口

保留命令面板中的停止命令，新增在任务运行期间可见的插件侧按钮。点击后：

- 立即 Abort 下载或执行；
- Windows 使用 `taskkill /T /F` 终止整个进程树；
- macOS 终止转写进程组，并保留兜底的子进程终止；
- 当前记录走可重试失败路径，不写入已同步状态；后续同步记录继续处理。

## 验证

- 纯函数：体验/正式/过期/免费四种绑定额度和到期排序。
- API：`/bind` 无效码、额度拒绝、体验有效时第二/第三码成功。
- 插件：裸 4xx 可恢复服务端错误体、阶段/心跳文案、停止按钮与 Windows 进程树策略。
- 安装脚本：版本标记、阶段/心跳标记、Windows/macOS 语法与现有安装器契约。

## 风险与回退

动态访问收敛涉及云函数部署，但本任务只提交代码和测试；部署须单独批准。若新进度协议无法读取，插件回退到旧文本进度；暂停失败不把记录标记为成功。
