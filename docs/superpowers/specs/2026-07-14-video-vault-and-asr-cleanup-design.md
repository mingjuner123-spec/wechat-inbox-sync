# 视频平台原视频保存与 ASR 尾部清理设计

## 目标

抖音、小红书和 B 站的视频链接开启“保存原始音视频到本地”后，只保存实际含视频轨的原视频；同时清除本地 ASR 在正文结尾产生的重复短句、字幕署名和类似幻觉文本。

## 原视频保存

视频平台不再把转写使用的音频 URL 当作附件来源。转写记录保存所有媒体候选，并区分视频候选；附件保存时逐个下载并检查 MP4/WebM 容器是否有视频轨。没有视频轨的候选直接跳过，不能被命名为 `.mp3` 并保存。

抖音和小红书复用现有详情接口、页面和隐藏渲染收集到的候选流，按实际轨道决定是否保存。B 站继续使用 DASH 音频做转写，同时额外请求 progressive `durl` 播放地址作为原视频候选；只有该地址下载后确实含视频轨才保存。因登录、限流、链接过期、DRM 或平台只返回音频而没有有效视频时，不保存附件，笔记保留转写并给出“未取得原视频”的状态。

## ASR 尾部清理

本地转写完成后，在写入笔记前运行纯文本清理器。它只从尾部回退：删除连续重复的相同短句（包括少于六字的重复）、字幕/翻译/署名行，以及紧邻这些异常尾巴的短英文或“画面”类噪声；正文中间的重复不会被修改。清理器没有删掉任何正文时返回原文本。

## 验证

新增测试覆盖：音频 MP4 不可作为视频附件、后续视频候选会被保存为 `.mp4`、没有视频候选返回专用错误、B 站 progressive 视频候选提取，以及用户提供的 ASR 幻觉尾部样本。保留现有 Pro 门禁、转写和市场包回归测试。

## 边界

不绕过 DRM、登录或平台访问限制；不下载历史笔记；不把 DASH 的独立视频片段伪装成含音频的完整视频。原视频只对新同步记录生效，原有错误附件需用户删除后重新同步。
