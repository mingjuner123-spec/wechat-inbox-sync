# Pro 原始音视频保存设计

## 目标

将“保存原始音视频到本地”从基础设置移入“Pro 高级功能”，并将它变为受云端 Pro 权益保护的能力。

## 范围

- 设置界面：开关只出现在“Pro 高级功能”中的 Pro 状态面板，说明其仅供有效 Pro 使用。
- 开关操作：用户尝试开启时，插件通过已绑定的小程序绑定码刷新并校验云端权益；校验失败则保持关闭并提示原因。
- 运行时：即使历史本地设置仍为开启，实际下载、写入 Vault 前也强制刷新云端 Pro 权益。无权限、未绑定、过期或请求失败时跳过附件保存，不影响已取得的转写或笔记同步。
- 不改变：默认值仍为关闭；附件目录、命名方式、下载错误处理和 Markdown 本地链接保持现状。

## 设计

现有 `getProFeatureAccessStatus` 和 `ensureProFeatureAccess` 已统一处理绑定码、兑换码及过期状态。扩展 `ensureProFeatureAccess` 使调用者可要求 `forceRefresh`，以便原始音视频保存这个付费写入动作每次均查询云端，而不是只接受本地缓存。

设置面板将原有基础区的 `Setting` 删除，并在 `proPanel` 中加入同一开关。开启事件先调用强制云端校验，成功才持久化 `saveOriginalMediaEnabled: true`；失败时写回 `false`、重绘界面并显示现有 Pro 错误信息。

`saveSourceMediaAttachment` 是唯一的下载写盘入口。它在检查本地开关后调用强制 Pro 校验；校验异常直接返回原始 record，故不会把 Pro 状态问题转化为同步失败，也不会生成空附件错误。

## 验收与测试

`tests/plugin-main-ai.test.js` 增加两类断言：

1. 付费校验失败时，即使保存开关为 true，`saveSourceMediaAttachment` 不下载、不写盘且返回原 record。
2. 有效 Pro 且强制刷新成功时，附件按原路径写入；测试确认校验调用指定 `forceRefresh`。

现有市场包测试、插件主测试和语法检查必须通过。发布时升版本并按现有 GitHub tag 工作流生成插件市场资产。

## 风险与边界

每个启用了该功能且有媒体直链的记录会额外请求一次云端权益状态；这是为了保证 Pro 权益不只依赖本地缓存。网络失败时采取保守策略：不保存原媒体，但仍保留转写和同步结果。
