# 体验 Pro 绑定提示与本地转写可控性实施计划

> 仅修改公开插件发布源；不部署、不发布。体验 Pro 的三绑定额度与到期动态失效另由私有云函数任务实现。

## 1. 先写失败测试

- 为 `/bind` 的裸 4xx 增加覆盖：只有服务端明确 `Invalid bind code` 才显示“绑定码无效”；额度或无法取回错误体时不得误报无效。
- 为本地 ASR 日志增加覆盖：解析 `progressStage`、`progressHeartbeatAt`、`progressPid` 与阶段开始时间；损坏/旧日志可安全降级。
- 为进度文案增加覆盖：准备、分段、首段转写、陈旧心跳的可区分文案与运行时长。
- 为脚本版本检查增加覆盖：旧 v2 日志脚本必须要求更新；带心跳/PID 标记的 Windows、macOS 脚本才为可用。
- 为停止策略增加覆盖：Windows 使用 `taskkill /PID <pid> /T /F` 并有 `kill()` 兜底；macOS 只在独立进程组时以负 PID 终止。
- 为暂停语义增加覆盖：当前记录不会标记 `/synced`，后续记录仍可继续，停止产生可重试失败。

## 2. 插件逻辑

- 将 `/bind` 的 HTTP 状态异常单独回读为 Node HTTP JSON，并映射结构化错误码；保留绑定码输入，不改变有效码绑定规则。
- 扩展 ASR 日志解析与进度去重键，心跳变化必须刷新 UI。
- 将本地阶段传入同步进度展示；心跳陈旧时显示可暂停重试的提示。
- 添加仅在本地转写运行期间显示的暂停入口，保留命令面板入口。
- 以平台安全方式终止进程树；停止后走现有可重试失败路径。

## 3. 安装器协议

- Windows 和 macOS 转写脚本均在分段、转写期间轮询原生子进程，5 秒写入实际 PID、开始时间、心跳和阶段。
- 增加版本标记，让已安装旧脚本触发既有“安装/修复”流程；不创建额外目录。
- 保持现有输出、分段、重复文本保护与清理行为。
