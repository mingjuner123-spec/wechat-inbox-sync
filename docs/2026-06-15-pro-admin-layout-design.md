# Pro 管理版面调整设计

## 目标

把 `Pro 管理` 页改成更适合日常运营查看的结构，减少“有效用户”和“历史/失效数据”混在一起的情况。

## 本次调整

1. `Pro 用户` 表只展示当前仍有效的 Pro 用户。
2. 原来状态为 `expired` 的用户单独放到 `已到期用户` 表，并在界面文案中直接显示中文“已到期”。
3. `兑换码列表` 默认只展示仍可兑换的兑换码。
4. 已兑换、已禁用、或其他不可再用的兑换码统一放到 `已兑换 / 已失效兑换码` 表。

## 范围

- 只调整 `admin-console` 前端页面的分组、文案和渲染。
- 不修改后端接口和数据库结构。

## 数据规则

- `Pro 用户`：`proStatus === 'active'`
- `已到期用户`：`proStatus === 'expired'`
- `可用兑换码`：`status === 'active'` 且 `redeemedCount < maxRedemptions`
- `已兑换 / 已失效兑换码`：不属于“可用兑换码”的其他记录

## 验收

- 打开 `Pro 管理` 页后，能同时看到独立的有效 Pro、已到期用户、可用兑换码、已兑换/已失效兑换码四块内容。
- 页面中不再直接向运营展示英文 `expired`。
