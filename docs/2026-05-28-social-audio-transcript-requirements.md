# 社交平台口播/音频文案提取需求

日期：2026-05-28

## 目标

用户在小程序里保存小宇宙、B站、抖音、小红书视频类链接后，Obsidian 插件同步时尽量提取“视频/音频里真实说了什么”，并写入 Obsidian。

本功能不以平台页面正文、标题、tag、封面图、评论等信息为核心，也不把这些内容作为主要输出。

## 范围

第一版支持：

1. 小宇宙单集链接：提取音频地址，转写为音频文案。
2. B站视频链接：优先提取已有字幕；没有字幕时尝试提取音频地址并转写。
3. 抖音视频链接：只输出口播/音频文案，不保存页面脚本、正文或 tag。
4. 小红书视频类链接：只在识别到视频源时进入口播转写；小红书图文笔记仍走原有图文保存逻辑。

## 转写优先级

1. 直接字幕：如果平台页面或接口能取到字幕，直接保存字幕，不消耗 AI API。
2. 已配置云端 ASR：复用插件已有的豆包、阿里云、腾讯云转写配置。
3. 本地转写：提供 Windows 一键准备脚本，安装 whisper.cpp、ffmpeg 和 small 模型后由插件调用本地命令。

## 输出格式

Markdown 只保留必要信息：

```md
原始链接：...

## 口播/音频文案

...
```

失败时写明失败原因，不把平台正文或 tag 当作口播文案。

## 非目标

- 不做平台正文、tag、封面、内页图的新增提取。
- 不影响现有小红书图文笔记功能。
- 不承诺所有平台视频都能无 API 转写。
- 不把本地模型文件直接打进插件包；通过安装脚本下载，避免插件体积过大。

## 验收标准

- 小红书图文现有测试保持通过。
- B站字幕 JSON 可以被解析成纯文案。
- 小宇宙页面中的音频地址可以被识别。
- 抖音和小红书视频页面中的视频/音频地址可以进入转写链路。
- 当没有字幕且未配置转写方案时，笔记中显示清晰失败原因。
- 本地转写方案有可执行的 Windows 安装脚本和默认命令。
